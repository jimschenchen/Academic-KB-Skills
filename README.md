# academic-kb: academic version of Karpathy LLM Wiki

A full-pipeline skill for managing an Obsidian-based academic knowledge base: **ingest в†’ read в†’ compile в†’ query в†’ lint**. Supports both academic papers (PDF) and articles (blogs, tech reports, white papers, social media).

Inspired by:
- [Karpathy's knowledge base workflow](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) вЂ” the core idea of an ingest в†’ compile в†’ query в†’ lint pipeline
- [Paperwise (sjqsgg)](https://github.com/sjqsgg/Paperwise) вЂ” dual-column HTML annotation design with 5-color highlight system

## Installation

Drag `academic-kb.skill` into a Claude Desktop chat window to install. Alternatively, copy the `academic-kb/` folder into your `.claude/skills/` directory.

**Recommended: use with [Obsidian](https://obsidian.md).** All paper notes and topic files generated by this skill use Obsidian Flavored Markdown (wikilinks, callouts, frontmatter properties) and rely on the Dataview plugin for the Paper Database's auto-query and aggregation views. Open the vault root as an Obsidian vault to get full bidirectional linking, graph visualization, and Dataview dashboards.

## Quick Start

### 0. Ingest вЂ” Import a new paper

```
/kb ingest papers/AlphaEvolve/AlphaEvolve.pdf
```

You can also give it a paper name, arXiv ID, or URL вЂ” it will download the PDF for you:

```
/kb ingest 2502.13131
/kb ingest "AlphaEvolve: A coding agent for scientific discovery"
/kb ingest https://arxiv.org/abs/2502.13131
```

**What it does:**
- Downloads the PDF if given a title, arXiv ID, DOI, or URL (confirms with you first)
- Parses PDF, extracts metadata and full text
- Generates structured bilingual paper note (12 sections 0-11, each with Chinese + English in parallel, + complete frontmatter)
- Automatically leverages HTML annotation from `/kb read` if available
- **Auto deep-read:** for long/complex papers (в‰Ґ12 pages, survey, high-relevance), creates per-section deep-read notes in `sections/` with insights, key quotes, and cross-references
- Suggests topic classification, updates corresponding topic files
- Embeds 5-10+ wikilinks throughout the note (to topics and other papers)
- Appends to kb-log

**Flags:** `--deep` (force section notes), `--no-deep` (skip section notes)


**What it does:**
- Fetches the web page (or accepts pasted text)
- Generates a 5-section article note: Core Argument, Key Insights, Evidence & Examples, Related Papers, My Take
- Identifies papers mentioned in the article and links to vault entries
- Flags papers worth ingesting that aren't in the vault yet
- Classifies into topics and updates topic files
- Optionally saves a snapshot.html to prevent link rot

### 1. Read вЂ” Annotate a paper

```
/kb read papers/AlphaEvolve/AlphaEvolve.pdf
```

**What it does:**
- Parses the full PDF, analyzes argument structure paragraph by paragraph
- Generates dual-column HTML: original text + 5-color highlights on the left, per-paragraph annotations on the right (paragraph function, logical role, rhetorical technique)
- Bottom section: argument structure overview + one-click APA/BibTeX copy
- HTML opens directly in any browser

**Question-driven mode:**
```
/kb read paper.pdf --questions "Q1: Key innovation? Q2: How does it compare to baselines?"
```
Switches to per-question coloring, generates a Q&A Worksheet at the bottom.

**Tip:** Run `/kb read` after ingest to generate HTML deep-reading annotation. Or run it before ingest вЂ” the paper note quality will be higher if an annotation exists.

### 2. Compile вЂ” Synthesize a topic

```
/kb compile Agent Memory
```

**What it does:**
- Reads key sections from all paper notes, article notes, and section deep-read notes under that topic
- Presents 3-5 cross-source insights for your review before writing
- Updates the topic file: framework comparison tables, convergent conclusions, tensions/debates, ranked open questions
- Performs backlink audit (ensures bidirectional links are complete)
- Updates paper counts in Topics Index.md

```
/kb compile --all    # compile all topics
```

### 3. Query вЂ” Cross-corpus research questions

```
/kb query "What's the intersection between Agent Memory and Agent RL?"
```

**What it does:**
- Finds relevant content from topic files, paper notes, article notes, and section deep-read notes (never from general knowledge)
- Synthesizes an answer with `[[wikilink]]` citations for every claim
- Saves to `queries/` folder; strong answers can be promoted into topics

### 4. Lint вЂ” Health check

```
/kb lint
```

**Checks for:**
- Orphan papers/articles (topics field points to non-existent topics)
- Dead wikilinks (topic files reference non-existent papers or articles)
- Missing frontmatter (missing title/authors/topics etc.)
- Missing backlinks (paper/article assigned to a topic but not listed in topic file)
- Stale counts (Topics Index.md counts out of date)
- Dead article URLs (source links no longer accessible, optional deep check)

Proposed fixes are listed per issue; only applied with your permission.

### 5. Log вЂ” View operation history

```
/kb log              # last 20 entries
/kb log ingest       # ingest events only
/kb log 2026-04      # April 2026 only
```

### Folder Details & Relationships

**`papers/`** вЂ” One subfolder per paper, named by the paper's original full title. Each contains the PDF source, a structured 12-section bilingual note (background, problem, methods, results, etc.), an optional HTML annotation from `/kb read`, and optionally a `sections/` subdirectory with per-section deep-read notes for long papers. The paper note's `topics` frontmatter field links it to `topics/` files. Wikilinks in the note body connect to other papers and topics.

**`articles/`** вЂ” One subfolder per article (blog, tech report, social media post). Each contains a 5-section note (core argument, key insights, evidence, related papers, personal take) and optionally a web page snapshot. Like papers, the `topics` frontmatter links articles to `topics/` files, and `recommended_papers` creates bridges to `papers/`.

**`topics/`** вЂ” Each file synthesizes all papers and articles tagged with that topic. Not a simple list вЂ” it provides framework comparison tables, cross-paper patterns, tensions/debates, and ranked open questions. The `Topics Index.md` serves as the central hub with paper counts and a mermaid dependency graph. Topic files link back to individual papers/articles via wikilinks.

**`queries/`** вЂ” Saved answers to cross-corpus research questions. Each query file cites specific papers, articles, and topics via wikilinks. Strong query answers can be promoted into new topics or merged into existing ones.

**`kb-log.md`** вЂ” Append-only audit trail of all operations (ingest, compile, query, lint) with timestamps, enabling you to track what was added or changed and when.

**How they connect:**

Every paper and article is linked to 1-5 topics via frontmatter. Topic files aggregate insights from all their constituent sources. Queries draw from all three and file back into the system. The graph grows denser with every ingest.
