# academic-kb: academic version of Karpathy LLM Wiki

A full-pipeline skill for managing an Obsidian-based academic knowledge base: **ingest → read → compile → query → lint**. Supports both academic papers (PDF) and articles (blogs, tech reports, white papers, social media).

Inspired by:
- [Karpathy's knowledge base workflow](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) — the core idea of an ingest → compile → query → lint pipeline
- [Paperwise (sjqsgg)](https://github.com/sjqsgg/Paperwise) — dual-column HTML annotation design with 5-color highlight system

## Installation

Drag `academic-kb.skill` into a Claude Desktop chat window to install. Alternatively, copy the `academic-kb/` folder into your `.claude/skills/` directory.

**Recommended: use with [Obsidian](https://obsidian.md).** All paper notes and topic files generated by this skill use Obsidian Flavored Markdown (wikilinks, callouts, frontmatter properties) and rely on the Dataview plugin for the Paper Database's auto-query and aggregation views. Open the vault root as an Obsidian vault to get full bidirectional linking, graph visualization, and Dataview dashboards.

## Quick Start

### 0. Ingest — Import a new paper

```
/kb ingest papers/AlphaEvolve/AlphaEvolve.pdf
```

You can also give it a paper name, arXiv ID, or URL — it will download the PDF for you:

```
/kb ingest 2502.13131
/kb ingest "AlphaEvolve: A coding agent for scientific discovery"
/kb ingest https://arxiv.org/abs/2502.13131
```

**What it does:**
- Downloads the PDF if given a title, arXiv ID, DOI, or URL (confirms with you first)
- Parses PDF, extracts metadata and full text
- Generates structured paper note (10 standard sections + complete frontmatter)
- Automatically leverages HTML annotation from `/kb read` if available
- Suggests topic classification, updates corresponding topic files
- Embeds 5-10+ wikilinks throughout the note (to topics and other papers)
- Appends to kb-log

### 0b. Ingest Article — Capture a blog, report, or social media post

```
/kb ingest article https://karpathy.github.io/2025/01/llm-os/
/kb ingest article "Karpathy 关于 LLM OS 的博客"
```

For content that's hard to scrape (微信公众号, 小红书), paste the text directly:

```
/kb ingest article [paste content here]
```

**What it does:**
- Fetches the web page (or accepts pasted text)
- Generates a 5-section article note: Core Argument, Key Insights, Evidence & Examples, Related Papers, My Take
- Identifies papers mentioned in the article and links to vault entries
- Flags papers worth ingesting that aren't in the vault yet
- Classifies into topics and updates topic files
- Optionally saves a snapshot.html to prevent link rot

### 1. Read — Annotate a paper

```
/kb read papers/AlphaEvolve/AlphaEvolve.pdf
```

**What it does:**
- Parses the full PDF, analyzes argument structure paragraph by paragraph
- Generates dual-column HTML: original text + 5-color highlights on the left, per-paragraph annotations on the right (paragraph function, logical role, rhetorical technique)
- Bottom section: argument structure overview + one-click APA/BibTeX copy
- HTML opens directly in any browser

**Question-driven mode:**
```
/kb read paper.pdf --questions "Q1: Key innovation? Q2: How does it compare to baselines?"
```
Switches to per-question coloring, generates a Q&A Worksheet at the bottom.

**Tip:** Run `/kb read` after ingest to generate HTML deep-reading annotation. Or run it before ingest — the paper note quality will be higher if an annotation exists.

### 2. Compile — Synthesize a topic

```
/kb compile Agent Memory
```

**What it does:**
- Reads key sections from all paper notes and article notes under that topic
- Presents 3-5 cross-source insights for your review before writing
- Updates the topic file: framework comparison tables, convergent conclusions, tensions/debates, ranked open questions
- Performs backlink audit (ensures bidirectional links are complete)
- Updates paper counts in Topics Index.md

```
/kb compile --all    # compile all topics
```

### 3. Query — Cross-corpus research questions

```
/kb query "What's the intersection between Agent Memory and Agent RL?"
```

**What it does:**
- Finds relevant content from topic files, paper notes, and article notes (never from general knowledge)
- Synthesizes an answer with `[[wikilink]]` citations for every claim
- Saves to `papers_queries/` folder; strong answers can be promoted into topics

### 4. Lint — Health check

```
/kb lint
```

**Checks for:**
- Orphan papers/articles (topics field points to non-existent topics)
- Dead wikilinks (topic files reference non-existent papers or articles)
- Missing frontmatter (missing title/authors/topics etc.)
- Missing backlinks (paper/article assigned to a topic but not listed in topic file)
- Stale counts (Topics Index.md counts out of date)
- Dead article URLs (source links no longer accessible, optional deep check)

Proposed fixes are listed per issue; only applied with your permission.

### 5. Log — View operation history

```
/kb log              # last 20 entries
/kb log ingest       # ingest events only
/kb log 2026-04      # April 2026 only
```

## Vault Structure

```
vault-root/
├── papers/
│   ├── [Paper Title]/
│   │   ├── [Paper Title].md           ← paper note (Obsidian)
│   │   ├── [Paper Title].pdf          ← original PDF
│   │   └── annotation.html            ← optional, from /kb read
│   └── Paper Database.md              ← Dataview auto-query dashboard
├── articles/
│   ├── [Article Title]/
│   │   ├── [Article Title].md         ← article note (5-section)
│   │   └── snapshot.html              ← optional, web page snapshot
│   └── Article Database.md            ← Dataview auto-query dashboard
├── papers_topics/
│   ├── [Topic Name].md                ← topic synthesis (papers + articles)
│   └── Topics Index.md                ← central hub + mermaid graph
├── papers_queries/                    ← cross-corpus research Q&A
│   └── [YYYY-MM-DD] [Question].md
└── kb-log.md                          ← append-only audit log
```

## Workflow

```
/kb ingest paper.pdf           → paper note + topic classification
/kb ingest article <url>       → article note + topic classification
      ↓ (optional deep reading for papers)
/kb read paper.pdf             → HTML annotation (open in browser)
      ↓ (after accumulating sources)
/kb compile "Topic Name"       → topic synthesis (papers + articles)
      ↓
/kb query "question"           → cross-corpus research Q&A
      ↓ (periodically)
/kb lint                       → health check + fix
```

## Recommended Obsidian Plugins

- **Dataview** (required) — Paper Database.md and Topics Index.md depend on it
- **Graph View** (built-in) — visualize the wikilink network between papers and topics
- **Templater** — optional, for quickly scaffolding new paper notes
